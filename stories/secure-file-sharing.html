The Problem

Organizations constantly struggle with securely sharing large files without compromising on data integrity, availability, and access control. Traditional methods involve:

  • Lack of Encryption: Files sent via email or standard FTP are often intercepted or exposed.
  • Scalability Limits: On-premise storage servers run out of space rapidly.
  • Data Redundancy: Need to ensure that uploaded objects are safe and consistently accessible.
  • Tracking: Difficulty monitoring who uploaded what, when it was sent, and to whom.

The Architecture & Solution

The Secure File-Sharing Web App tackles these challenges by shifting the entire architecture onto AWS. Implementing a microservice-like serverless approach, components are decoupled for supreme resilience and scalability.

Web Server

Python Flask running on EC2 instances for resilient computing.

Storage

AWS S3 stores objects with generated presigned URLs for secure temporary access.

Database

AWS RDS (MySQL) logs metadata: file ID, generated S3 key, presigned URL, and recipient email list.

Notifications

AWS Lambda triggered via API to send tracking emails leveraging AWS Simple Email Service (SES).

System Workflow

The system utilizes a Python Flask orchestrator to manage the flow of uploaded objects securely into AWS S3, tracking state in MySQL, and invoking Event-driven serverless Lambda functions.

Key Features

Serverless Notifications

Decoupled architecture utilizing AWS Lambda and SES ensures smooth, non-blocking asynchronous email delivery.

Secure Storage via S3

Instead of passing raw files, the system generates time-bound Presigned URLs for S3 objects, dramatically increasing data safety.

Scalable Compute

Hosted on AWS EC2, capable of sitting behind an Application Load Balancer with Auto Scaling potential.

Relational Tracking

Fully managed AWS RDS (MySQL) handles relational database tracking of every uploaded item, enabling auditing.

Impact & Results

Security & Scalability Achieved

  • Decoupled file uploads from email dispatch entirely, minimizing wait times.
  • Implemented IAM Roles/Security Groups using the principle of least privilege.
  • Reduced risk of data interception via presigned temporary URL access.
  • Created a highly extensible boilerplate for secure enterprise cloud applications.
